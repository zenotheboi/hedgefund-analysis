BioPharmCatalyst small-molecule pricing analysis

Cumulative abnormal return (CAR, T-2..T+2 vs. XBI) around FDA Approved or CRL (Complete Response Letter) catalysts, small-molecule/peptide drugs only, filtered from the BioPharmCatalyst scrape (Tejas1415/Web-Scrape-Stock-Ticker-and-Company-Name-Datasets).

Coverage caveats, read before using these numbers: the source file covers 2009-07-05 to 2020-01-14 only — not the trailing 10 years as originally hoped; it is a static scrape, stale by ~6 years. Of 462 unique small-molecule/peptide Approved-or-CRL events (363 rows after PubChem filtering), 249 (68.6%) have priceable Yahoo Finance history — the other 114 fail because 55 tickers were delisted/acquired long enough ago that Yahoo has purged their price history entirely (e.g. ARNA, CLVS, SGEN, MYL, AGN — all real M&A exits), not because of a pipeline bug.

Proposal: separating the full dataset into trial phases

The user's core question — the raw file's stage column mixes phase labels with regulatory-decision labels in one field, with no explicit phase-of-pivotal-trial link for Approved/CRL rows.

What the "Approved or CRL" column actually contains

It's not an outcome column and a phase column layered together by accident — it's a single catalyst-type field with 13 distinct values in the cleaned data:

ValueCountWhat it means
Phase 1 / 1b / 1-2259early readout, not yet pivotal
Phase 2 / 2a / 2b566mid-stage readout
Phase 2/323seamless/adaptive trial readout
Phase 3689pivotal-trial readout
NDA / BLA Filing, PDUFA10regulatory-process milestones, not trial results
Approved513terminal regulatory decision (positive)
CRL120terminal regulatory decision (negative)

Why Approved/CRL already is effectively "post-Phase-3"

FDA approval decisions are made almost entirely on pivotal Phase 3 data (accelerated-approval-off-Phase-2 is the exception, not the rule). So the Approved/CRL slice used above is already a reasonable proxy for "Phase 3 pivotal outcome" — this is the same population the old EDGAR-matched pipeline was trying to approximate at n=47, just reached here directly through a curated regulatory-decision date instead of guessing a trial-completion-to-filing match. That's the main reason this source is a real improvement, not just a different dataset.

Recommended linking scheme, to get an explicit phase per event

  1. Primary (free, no external calls): for every Approved/CRL row, look back within this same CSV for the most recent prior row sharing Ticker + a fuzzy-matched Drug Name (strip parentheticals/trial-name suffixes the same way the PubChem retry step did) whose status is Phase 1/2/3. That row's phase is the pivotal phase. Cheap, deterministic, no rate limits — the same trick that made PubChem name-matching recover 267 of 462 drugs here.
  2. Validation on a sample: cross-check ~30 linked pairs against Drugs@FDA (openFDA, free) by brand/generic name — it returns submission type (NDA/BLA) and approval history but not trial phase directly, so it validates the drug identity match, not the phase itself.
  3. Fallback for unlinked rows: ClinicalTrials.gov search by intervention name + phase=PHASE3 + status Completed, dated before the Approved/CRL catalyst date — same idea as the CTOD/EDGAR pipeline in the other project, but only needed for whatever step 1 can't resolve, which should be a minority.

What this unlocks beyond the Approved/CRL slice done here

The 1,547 Phase 1/1b/1-2/2/2a/2b/2-3/3 rows have no explicit success/failure label — only free-text Catalyst Description ("met its primary endpoint" vs. "did not meet…"). A simple keyword classifier on that text (same result-language approach used for EDGAR filing scoring in the other project) would let mid-phase readouts be priced the same way as Approved/CRL here — extending this analysis to Phase 1/2 catalysts, not just terminal regulatory decisions.
